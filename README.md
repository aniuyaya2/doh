Cloudflare Snippets 环境比常规 Workers 更加严格：它每次执行最多只允许发起 5 个子请求（fetch）。
在原版代码的 ⁠queryDns⁠ 函数中，为了兼容各种上游 DoH，作者写了一个“多重请求头重试”的循环（最多尝试 4 次）。当我们在页面上点击解析时（默认同时查询 A、AAAA 和 NS 记录），代码会同时发起 3 个查询。如果触发了重试逻辑，瞬间就会产生 12 次以上的 fetch 请求，直接打爆 Snippets 的并发限制，导致 Cloudflare 强制熔断并返回 500 错误。
为了彻底解决这个问题，我对底层的 ⁠queryDns⁠ 请求逻辑进行了极限精简，去掉了危险的重试循环，直接指定最通用的请求头。
